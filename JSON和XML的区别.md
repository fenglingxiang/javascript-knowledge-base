## JSON和XML的区别

- XML更像一个文档，功能强大、灵活，但相对复杂和冗长，它专注于使用标签来描述数据及其结构
- JSON更接近原数据本身，轻量、简洁、易于解析，它主要是通过键值对或是数组的形式表示数据

| 特性             | XML                                                          | JSON                                                         |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **全称与含义**   | 可扩展标记语言<br />专注于标记和描述对象                     | JavaScript对象表示法<br />专注于表示数据对象                 |
| **数据结构**     | 树形结构，通过标签嵌套实现                                   | 键值对和有序列表，源于js对象                                 |
| **语法格式**     | 通过开始标签和结束标签展示<br />需要根元素<br />属性需要引号 | 通过{}表示对象<br />[]表示数组<br />键名必须双引号           |
| **可读性**       | 结构清晰，但因为数据与标签混合比较冗长                       | 接近数据本身，简洁易于理解                                   |
| **解析难度**     | 相对复杂，需要专门的XML解析器                                | 极其简单，在js中可直接通过<br />JSON.parse()转译，其他语言也内置支持 |
| **命名空间**     | 支持，适合复杂、混合文档                                     | 不支持                                                       |
| **注释**         | 支持                                                         | 不支持                                                       |
| **数据类型**     | 所有内容都是字符串，需要额外转换                             | 支持原生的数字、字符串、布尔值、对<br />象、数组、null       |
| **典型应用场景** | 配置文件、文档存储、Web Service、企业级应用                  | Web APIs、NoSQL数据库、配置文件、前后端数据交换              |
| **扩展性**       | 较好，可自定义标签和结构                                     | 好，但结构相对固定                                           |
